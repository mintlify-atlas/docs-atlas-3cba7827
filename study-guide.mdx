This study guide helps you navigate the MCP for Beginners curriculum efficiently. Use it to understand how all topics connect, choose a learning path that matches your experience level, and make the most of the sample projects.

<Info>
  The curriculum follows **MCP Specification 2025-11-25**. This study guide was last updated February 5, 2026.
</Info>

## Repository Overview

The Model Context Protocol (MCP) is a standardized framework for interactions between AI models and client applications. Initially created by Anthropic, MCP is now maintained by the broader MCP community through the official GitHub organization.

This curriculum covers:
- **11 core modules** (00–11) in a sequential learning path from fundamentals to advanced topics
- **Hands-on labs** with complete solution code in multiple languages
- **Sample projects** — working MCP server and client implementations
- **Multi-language support** — documentation translated into 48+ languages

## Recommended Learning Paths

Choose the path that fits your goals:

<Tabs>
  <Tab title="Complete Beginner">
    Follow all modules in order from 00 to 11. Take time with Modules 00–02 to build a solid conceptual foundation before writing code.

    **Estimated time**: 40–60 hours

    **Suggested pace**:
    1. Modules 00–02: Spend 2–3 hours each building concepts
    2. Module 03: Work through sub-guides one at a time, running each example
    3. Modules 04–05: Read and experiment with topics that interest you
    4. Modules 06–09: Skim for community context; return when needed
    5. Module 11: Use as a capstone to tie everything together
  </Tab>
  <Tab title="Experienced Developer">
    If you already understand client-server architecture and REST APIs, you can move faster through the foundation modules.

    **Suggested path**:
    1. Skim Modules 00–01 for MCP-specific terminology
    2. Read Module 02 fully — security is critical
    3. Start hands-on with Module 03 sub-guides in your language of choice
    4. Dive directly into Module 05 topics that match your use case
    5. Use Module 11 as a production reference implementation
  </Tab>
  <Tab title="Language-Specific Focus">
    Explore the `samples/` directories within each module for your preferred language:

    | Language | Key Samples Location |
    |---|---|
    | C# | `03-GettingStarted/samples/csharp/`, `04-PracticalImplementation/samples/csharp/` |
    | Java | `03-GettingStarted/samples/java/`, `04-PracticalImplementation/samples/java/` |
    | JavaScript | `03-GettingStarted/samples/javascript/` |
    | TypeScript | `03-GettingStarted/samples/typescript/` |
    | Python | `03-GettingStarted/samples/python/`, Module 11 (FastMCP) |
    | Rust | `03-GettingStarted/samples/rust/` |

    Each sample directory contains its own README with setup instructions.
  </Tab>
  <Tab title="Security Focus">
    If your primary interest is securing MCP deployments:

    1. Read Module 02 in full, including the supplementary documents
    2. Complete the [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) hands-on labs
    3. Study the OWASP MCP Top 10 via the [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/)
    4. Review Module 05 sections on OAuth2, Entra ID, and advanced security
    5. Study Module 11 labs 02 (Row Level Security) and 10 (Deployment) for production patterns
  </Tab>
</Tabs>

## How to Use Sample Projects

Sample projects are organized by module and language:

<Steps>
  <Step title="Navigate to the sample directory">
    Each module's `samples/` folder contains language-specific subdirectories. For example, `03-GettingStarted/samples/python/` contains the Python MCP calculator server.
  </Step>
  <Step title="Read the local README">
    Every sample contains its own `README.md` with specific setup instructions, dependencies, and what the sample demonstrates.
  </Step>
  <Step title="Install dependencies">
    Follow the language-specific setup:
    - **TypeScript/JavaScript**: `npm install && npm start`
    - **Python**: `pip install -r requirements.txt`
    - **Java**: `mvn clean install && mvn spring-boot:run`
    - **C#**: Use the .NET CLI or Visual Studio
  </Step>
  <Step title="Run and explore">
    Start the server and connect a client. Use the MCP Inspector (Module 3.13) to debug and inspect the server's exposed tools, resources, and prompts.
  </Step>
  <Step title="Modify and experiment">
    Change tool definitions, add new tools, or swap transport mechanisms. The best learning happens when you break things and fix them.
  </Step>
</Steps>
