This module takes you from zero to a working MCP implementation. You will build servers, write clients, add LLM integration, configure IDE tooling, explore transport options, test your code, and deploy to the cloud.

<Note>
Think of MCP like a USB-C port for AI applications — it provides a standardized way to connect AI models to different data sources and tools, regardless of which language or framework you use.
</Note>

## What you'll build

By the end of this module you will have:

- A working MCP server with tools, resources, and prompts
- A client that connects to and invokes your server
- A client powered by an LLM that selects tools automatically based on natural language
- VS Code + GitHub Copilot wired up to your server
- An understanding of both stdio and Streamable HTTP transports
- Tests for your server logic
- A deployment strategy for local and cloud environments

## Official SDKs

MCP provides official SDKs for every major language. All examples in this module use these SDKs.

| Language | SDK | Maintainer |
|----------|-----|------------|
| TypeScript | [@modelcontextprotocol/sdk](https://github.com/modelcontextprotocol/typescript-sdk) | Anthropic |
| Python | [mcp (FastMCP)](https://github.com/modelcontextprotocol/python-sdk) | Anthropic |
| C# / .NET | [ModelContextProtocol](https://github.com/modelcontextprotocol/csharp-sdk) | Microsoft |
| Java | [mcp-java-sdk](https://github.com/modelcontextprotocol/java-sdk) | Spring AI |
| Kotlin | [kotlin-sdk](https://github.com/modelcontextprotocol/kotlin-sdk) | Anthropic |
| Rust | [rmcp](https://github.com/modelcontextprotocol/rust-sdk) | Anthropic |
| Swift | [swift-sdk](https://github.com/modelcontextprotocol/swift-sdk) | Loopwork AI |
| Go | [go-sdk](https://github.com/modelcontextprotocol/go-sdk) | Anthropic |

## Key takeaways

- Setting up an MCP development environment is straightforward with language-specific SDKs.
- MCP servers expose **tools** (functions), **resources** (data), and **prompts** (templates) through a unified protocol.
- MCP clients connect to servers and models to leverage extended capabilities.
- Testing and debugging are essential for reliable MCP implementations.
- Deployment options range from local development to cloud-based solutions.
